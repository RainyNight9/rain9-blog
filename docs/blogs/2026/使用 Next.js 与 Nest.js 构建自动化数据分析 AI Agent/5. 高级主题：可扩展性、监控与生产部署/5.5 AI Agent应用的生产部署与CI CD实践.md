# 5.5 AI Agent应用的生产部署与CI/CD实践

终于到了最后一步！你的应用在本地运行完美，现在要把这套由 Next.js、Nest.js、PostgreSQL 组成的复杂系统推向生产环境。

## 1. 容器化 (Docker 化) 生产部署

在第 2 章中我们用 Docker Compose 跑起了数据库。现在我们要把 Nest.js 后端也打包成镜像。

```dockerfile
# backend/Dockerfile.prod
FROM node:18-alpine AS builder
WORKDIR /app
COPY package.json pnpm-lock.yaml ./
RUN npm install -g pnpm && pnpm install
COPY . .
RUN pnpm build

FROM node:18-alpine AS runner
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY package.json ./
ENV NODE_ENV=production
CMD ["node", "dist/main"]
```

前端 Next.js 也可以类似打包，或者直接部署到 Vercel 享受全球 CDN。

## 2. CI/CD 流水线 (持续集成与持续部署)

利用 GitHub Actions 或 GitLab CI，实现代码提交后的自动化流程：

1. **Lint & Test**: 运行 ESLint 和 Jest 单元测试。
2. **Build**: 构建前端和后端项目，确保没有 TypeScript 编译错误。
3. **Docker Build & Push**: 构建 Docker 镜像并推送到镜像仓库 (如 Docker Hub 或 AWS ECR)。
4. **Deploy**: 触发生产服务器拉取最新镜像并重启服务 (如 Kubernetes 滚动更新或 Docker Swarm)。

## 3. 安全性与密钥管理的最佳实践

在生产环境中，**绝对不能**将 `.env` 文件打包进 Docker 镜像！

- 使用云提供商的密钥管理服务 (如 AWS Secrets Manager, Vercel Environment Variables, Kubernetes Secrets) 动态注入。
- 确保你的 OpenAI API Key 开启了调用额度上限，防止服务被 DDoS 攻击导致破产。
