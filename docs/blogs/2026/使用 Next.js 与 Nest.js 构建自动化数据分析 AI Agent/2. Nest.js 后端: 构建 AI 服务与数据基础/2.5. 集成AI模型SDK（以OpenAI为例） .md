# 2.5. 集成 AI 模型 SDK（以 OpenAI 为例）

在上一节中，我们完成了环境变量的安全配置。本节我们将正式集成 OpenAI SDK，构建与大语言模型（LLM）交互的基础能力。我们将遵循 Nest.js 的模块化设计原则，将 AI 服务封装为独立的模块，以便于维护和测试。

## 1. 安装依赖

首先，我们需要安装 OpenAI 官方提供的 Node.js SDK。

```bash
pnpm add openai
```

## 2. 封装 OpenAI 模块

为了保持代码的整洁和可复用性，我们不建议直接在业务逻辑中实例化 OpenAI 客户端。相反，我们将创建一个专门的 `OpenAIModule`。

### 2.1 创建模块与服务

使用 Nest CLI 快速生成模块和服务：

```bash
nest g module openai
nest g service openai
```

### 2.2 实现 OpenAIService

修改 `src/openai/openai.service.ts`，利用 `ConfigService` 读取我们在上一节配置的 API Key，并初始化 OpenAI 客户端。

```typescript
// src/openai/openai.service.ts
import { Injectable, Logger } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';
import OpenAI from 'openai';

@Injectable()
export class OpenAIService {
  private readonly openai: OpenAI;
  private readonly logger = new Logger(OpenAIService.name);

  constructor(private readonly configService: ConfigService) {
    const apiKey = this.configService.get<string>('OPENAI_API_KEY');
    
    if (!apiKey) {
        this.logger.error('OPENAI_API_KEY is not defined');
        throw new Error('OPENAI_API_KEY is not defined');
    }

    this.openai = new OpenAI({
      apiKey: apiKey,
    });
  }

  /**
   * 发送聊天请求
   * @param content 用户输入的内容
   * @param model 模型名称，默认为 gpt-3.5-turbo
   */
  async chat(content: string, model = 'gpt-3.5-turbo'): Promise<string> {
    try {
      const completion = await this.openai.chat.completions.create({
        messages: [{ role: 'user', content }],
        model: model,
      });

      return completion.choices[0]?.message?.content || '';
    } catch (error) {
      this.logger.error(`OpenAI API call failed: ${error.message}`);
      throw error;
    }
  }
}
```

### 2.3 导出服务

为了让其他模块（如 `AnalysisModule`）能够使用 `OpenAIService`，我们需要在 `OpenAIModule` 中将其导出。

```typescript
// src/openai/openai.module.ts
import { Module } from '@nestjs/common';
import { OpenAIService } from './openai.service';
import { ConfigModule } from '@nestjs/config';

@Module({
  imports: [ConfigModule], // 确保 ConfigModule 可用（如果是全局配置也可省略，但显式导入更安全）
  providers: [OpenAIService],
  exports: [OpenAIService], // 重要：导出服务供其他模块使用
})
export class OpenAIModule {}
```

## 3. 在业务模块中使用

现在我们可以在 `AnalysisModule` 中注入并使用 `OpenAIService` 了。

### 3.1 导入 OpenAIModule

修改 `src/analysis/analysis.module.ts`：

```typescript
// src/analysis/analysis.module.ts
import { Module } from '@nestjs/common';
import { AnalysisController } from './analysis.controller';
import { AnalysisService } from './analysis.service';
import { OpenAIModule } from '../openai/openai.module'; // 导入模块

@Module({
  imports: [OpenAIModule], // 添加到 imports
  controllers: [AnalysisController],
  providers: [AnalysisService],
})
export class AnalysisModule {}
```

### 3.2 调用 AI 服务

修改 `src/analysis/analysis.service.ts`，实现真正的 AI 分析逻辑。

```typescript
// src/analysis/analysis.service.ts
import { Injectable } from '@nestjs/common';
import { OpenAIService } from '../openai/openai.service';

@Injectable()
export class AnalysisService {
  constructor(private readonly openAIService: OpenAIService) {}

  async analyzeData(data: any): Promise<string> {
    // 构建 Prompt
    const prompt = `请作为数据分析师，分析以下数据并给出专业见解：\n${JSON.stringify(data)}`;
    
    // 调用 AI 服务
    return this.openAIService.chat(prompt);
  }

  // 保留原有方法兼容性
  getAnalysisResult(): string {
    return 'This is a detailed analysis result from the AnalysisService.';
  }
}
```

## 4. 验证

启动应用并调用相应的 API 接口，如果配置正确，你的 Nest.js 后端现在应该能够通过 OpenAI API 返回智能分析结果了。

```bash
pnpm start:dev
```

---

通过这种模块化的方式，我们将 AI 基础设施代码（OpenAI 模块）与业务逻辑代码（Analysis 模块）解耦，为后续扩展（如切换到其他 AI 模型）打下了良好的基础。
