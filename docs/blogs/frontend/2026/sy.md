# SY 项目经验

## 🚀 核心业务项目

### 1. 冬奥会出行保障与调度系统 (首汽约车)

**技术栈**：React 16, UmiJS, TypeScript, Ant Design, 高德地图 API
**项目描述**：为北京冬奥会量身定制的交通保障系统，承接奥组委官员与运动员从市区到场馆的专车调度服务。
**核心职责**：
*   **架构选型与搭建**：采用 **UmiJS + TypeScript** 技术栈搭建项目基座，设计了模块化开发规范，支撑了订单管理、运力监控、客诉处理等 10+ 个子模块的并行开发。
*   **配置化开发引擎**：设计并实现了一套基于 **JSON Schema** 的页面生成引擎，通过配置即可生成标准的 CRUD 页面与表单，将重复业务开发效率提升 **50%**。
*   **实时运力监控**：基于高德地图 API 与 WebSocket 开发**车辆实时监控中心**，实现了千级车辆位置的毫秒级刷新与轨迹回放，保障了冬奥期间的精准调度。
*   **复杂业务闭环**：打通了从基础信息录入、运力指派、行程监控到客诉处理的全链路业务闭环，确保了赛事期间零重大事故。

### 2. 出行大数据可视化监控大屏

**技术栈**：Vue / React, ECharts, 高德地图 API, WebSocket
**项目描述**：面向公司高层与运营中心的运力监控大屏，实时展示全国范围内的订单热力图、运力分布与营收数据。
**核心职责**：
*   **海量数据可视化**：解决了高并发数据下的前端渲染性能瓶颈，利用 **Canvas** 绘制万级车辆热力图，页面 FPS 稳定在 50+。
*   **多端适配**：实现了大屏系统在 PC 端、移动端小程序及监控大屏等多终端的自适应展示，满足了不同场景下的监控需求。
*   **决策支持**：通过多维度的图表联动分析，直观展示运力缺口与异常订单，为运营部门的实时运力调整提供了关键数据支撑。

### 3. 首汽约车 App 混合开发与 H5 矩阵

**技术栈**：Vue.js, Video.js, JSBridge, 高德地图 API
**项目描述**：负责乘客端与司机端 App 内核心 H5 业务模块的开发，涵盖积分商城、行程分享、车联网直播等高频功能。
**核心职责**：
*   **Hybrid 交互优化**：封装通用的 **JSBridge** 适配层，抹平了 iOS 与 Android 的原生能力差异（如定位、支付、分享），提升了 H5 页面在 App 内的加载速度与交互体验。
*   **车联网直播流**：基于 **Video.js** 实现了车内视频监控的实时直播功能，优化了弱网环境下的推流稳定性，保障了司乘安全。
*   **业务矩阵建设**：搭建了积分商城、企业用车服务等营销类 H5 页面，并通过组件化复用，快速支持了微信公众号（M 站）等多渠道的业务投放。

### 4. 乘客端与司机端小程序矩阵

**技术栈**：原生小程序, uni-app, Taro
**项目描述**：覆盖微信生态的轻量级打车入口，包括乘客快捷下单与司机招募纳新小程序。
**核心职责**：
*   **跨端重构**：主导将原本分散的原生小程序业务逐步迁移至 **uni-app** 跨端框架，实现了“一次开发，多端运行”（微信/支付宝/百度小程序），维护成本降低 **40%**。
*   **性能优化**：针对小程序启动速度与包体积进行深度优化，采用分包加载与图片懒加载策略，首屏加载时间缩短至 **1.5s** 以内。
*   **用户增长**：配合运营活动开发了裂变营销功能（如邀请有礼、拼团券），有效提升了小程序的日活（DAU）与司机注册转化率。
