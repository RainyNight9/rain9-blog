# AI Agent 项目经验 - 面试表达指南

## 1. 项目一句话介绍 (Elevator Pitch)

> **面试官：** 请介绍一下你最近做的 AI 项目。

**回答：**
“我参与核心开发的 **Coco App**，是一个**企业级的智能搜索与 AI Agent 平台**。
它不仅仅是一个对话机器人，而是基于 **RAG（检索增强生成）** 和 **MCP（模型上下文协议）** 构建的生产力工具。
它能够连接企业内部的各种数据源（如 GitHub, Notion, 数据库），具备**深度研究 (Deep Research)** 能力，可以自主拆解任务、调用工具并生成结构化的研究报告。”

---

## 2. 核心职责与技术亮点 (Key Responsibilities)

> **面试官：** 你在这个项目中主要负责什么？用到了哪些关键技术？

**回答：**
“我主要负责 **Agent 前端交互架构的设计与实现**，以及 **Tauri 桌面端与 Rust 后端的深度集成**。核心技术亮点包括：”

### A. 复杂流式交互设计 (Frontend Interaction)
*   **挑战**：Agent 的响应不是简单的文本流，而是包含了“思考过程 (CoT)”、“工具调用 (Tool Calls)”、“引用来源 (Citations)”等多维信息的混合流。
*   **解决方案**：
    *   我设计了一套**基于 SSE (Server-Sent Events) 的流式解析器**，能够实时拆分并处理 `thought`（思维链）、`tool_call`（工具状态）和 `content`（最终内容）。
    *   实现了 **增量渲染引擎**，让用户能实时看到 Agent 的“思考步骤”（如 `<Think />` 组件）和“行动轨迹”（如正在搜索数据库...），极大地提升了用户对 AI 的信任感和容忍度。

### B. 模型上下文协议 (MCP) 集成
*   **挑战**：传统的 Agent 对接外部工具需要写大量胶水代码，扩展性差。
*   **解决方案**：
    *   我们在项目中落地了 **MCP (Model Context Protocol)** 标准。
    *   我在前端实现了 **MCP 管理界面**，允许用户动态挂载/卸载数据源。
    *   通过标准化接口，Agent 可以自动发现并调用本地或远程的工具（如计算器、文件搜索），实现了工具生态的解耦。

### C. 跨平台混合架构 (Hybrid Architecture)
*   **挑战**：需要在桌面端提供高性能的本地能力（如读取本地文件索引）。
*   **解决方案**：
    *   采用 **React + Tauri (Rust)** 架构。前端负责 UI 状态管理，Rust 层作为系统桥梁。
    *   我封装了 `useChatActions` Hook，管理 Agent 的多阶段推理状态（意图识别 -> 工具选择 -> 数据获取），确保了前后端状态的严格同步。

## 3. 针对性面试问答 (Q&A Scenarios)

### Q1: 你们的 Agent 是如何解决“幻觉”问题的？
**回答：**
“我们主要通过 **RAG (检索增强生成)** 和 **引用溯源** 来解决。
在技术实现上，当 Agent 生成回答时，会携带引用的元数据。我在前端开发了 `<Citation />` 组件，会在回答的对应位置高亮显示引用来源（如‘参见文档 A 第 3 页’），用户点击即可跳转查看原始数据。这让 AI 的回答变得‘有据可查’。”

### Q2: 如何优化 Agent 的响应速度体验？
**回答：**
“大模型的推理确实有延迟，特别是开启深度思考模式时。我的优化策略是：
1.  **极致的流式渲染**：不等待完整句子，按字符级 (Token) 更新 UI。
2.  **可视化推理过程**：通过将 Agent 的 `loadingStep`（如正在分析意图、正在调用工具）可视化，把‘等待时间’转化为‘用户阅读 AI 思考过程的时间’，心理上降低了等待焦虑。”

### Q3: 你们的前端架构是如何支持多种 Agent 形态的？
**回答：**
“我们采用了 **Monorepo** 策略，将核心聊天能力抽离为 `@infinilabs/ai-chat` 和 `@infinilabs/chat-message` 独立包。
这样我们的一套核心代码，既能构建出类似 Perplexity 的**全屏深度搜索版**，也能构建出嵌入在 IDE 或业务系统右下角的 **Copilot 悬浮窗版**，实现了最大程度的代码复用。”

## 4. 总结升华 (Closing)

“通过 Coco 项目，我不仅仅掌握了 LLM API 的调用，更重要的是深入理解了 **AI Native 应用的交互范式**——即如何通过结构化的 UI（如思维链、工具卡片）来弥补大模型的黑盒特性，以及如何通过标准协议（如 MCP）来构建可扩展的 Agent 生态。”
